# 리서치 절차

## 참고 소스
- `https://library.gabia.com/`

## 볼 것
- 클라우드/IaaS 관련 주제
- SMB가 검색할 만한 문제
- 가격, 견적, 도입, 운영, 비용, 확장 관련 표현
- 가이드, 체크리스트, 비교, 사례형 콘텐츠 패턴

## 페르소나 질문
- C레벨: 비용, 매출, 의사결정, 투자 대비 효과
- 개발 팀장: 도입 난이도, 운영 부담, 사용 절차, 내부 설득 자료

## 후보 주제 만들기
- 먼저 8~12개를 만듭니다.
- 각 후보에 구매 여정, 페르소나, 고객 질문, 형식, CTA를 붙입니다.
- 최종 4~5개만 남깁니다.

## 선정 기준
- 고객 문제가 분명한가
- 검색 의도가 있는가
- 브랜드/IaaS 서비스와 연결되는가
- 세일즈 활용 가치가 있는가
- 제작 난이도가 지나치게 높지 않은가

## 승인 흐름
- 최종 일정 전에 담당 서비스 마케터 확인 필요 문구를 표시합니다.
- 사용자가 이미 승인했거나 초안만 요청한 경우에는 draft로 진행합니다.
